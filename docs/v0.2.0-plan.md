# TokenSquash v0.2.0 Plan

Status: development planning. v0.2.0 is intended to make real local turn
evidence easier to inspect before adding more deterministic codec rules.

## Product Goal

TokenSquash v0.2.0 should answer one practical question:

> Is this real redacted prompt/reply corpus clean, large, meaningful, and useful
> enough to justify the next codec or sidecar experiment?

The deterministic `ts1` prompt codec and `tr1` reply codec remain the source of
truth. The sidecar remains experimental and model-dependent.

## Main Work

### Real Corpus Scorecard

Add and harden:

```powershell
python -m tokensquash turns scorecard private-turns\real.redacted-turns.jsonl
```

The scorecard should summarize:

- turn count and corpus growth milestone
- original, squashed, and saved token totals
- prompt and reply savings
- adaptive pass-through rows
- raw wire loss turns
- privacy findings
- repeated path and field candidates
- alias-impact delta and break-even signal
- sidecar pass, watch, and fail counts when review evidence is present
- the next recommended action

### Corpus Growth Milestones

Use these milestones for real local corpora:

| Turns | Milestone | Meaning |
|---:|---|---|
| `< 10` | `seed` | Keep capturing. Do not infer patterns yet. |
| `10-24` | `smoke` | Good enough for a first local read. |
| `25-99` | `early_pattern` | Useful for finding repeated workflow language. |
| `100+` | `benchmark_ready` | Large enough for serious local benchmark evidence. |

These are evidence milestones, not product claims.

### Sidecar Meaning Evidence

Sidecar review rows should expose a clear reviewer decision:

- `pass`: no review flags
- `watch`: warnings, missing fields, generic summaries, or suspicious savings
- `fail`: high-risk rows or evaluation failures

Token savings alone are not success. Sidecar savings only become useful when the
review/gate/certification evidence shows meaning survived.

### Release-Tag Comparison

v0.2.0 should make it easy to compare current evidence against the last public
release. The preferred workflow is:

```powershell
python -m tokensquash turns scorecard private-turns\real.redacted-turns.jsonl --json --out private-turns\scorecards\current.json
python -m tokensquash turns scorecard-pack private-turns\real.redacted-turns.jsonl --out-dir private-turns\scorecard-pack --history private-turns\scorecards\v0.1.1.json
python -m tokensquash turns compare-scorecards private-turns\scorecards\v0.1.1.json private-turns\scorecards\current.json
python -m tokensquash turns scorecard-history private-turns\scorecards\v0.1.1.json private-turns\scorecards\current.json
```

The pack report uses schema `tokensquash.turns.scorecard.pack.v1` and writes
`scorecard.json`, `scorecard.md`, and optional `history.json` / `history.md`
artifacts into one ignored evidence directory.

The comparison report uses schema `tokensquash.turns.scorecard.compare.v1` and
returns `pass`, `watch`, or `fail` for release-review automation. A `fail`
means a hard-risk signal increased, such as privacy findings, sidecar failures,
or raw wire loss turns.

The history report uses schema `tokensquash.turns.scorecard.history.v1` and
summarizes multiple scorecards as a timeline with adjacent pass/watch/fail
comparison steps, net deltas, best/worst saved percent, and warnings.

## Acceptance Checklist

- `turns scorecard --json` reports schema `tokensquash.turns.scorecard.v1`.
- Markdown scorecard shows corpus milestone, savings, sidecar summary, repeated
  candidates, and recommendations.
- Scorecard works without sidecar evidence and reports sidecar status
  `missing`.
- Scorecard can summarize a saved `sidecar review` or `sidecar evaluate`
  report.
- Sidecar review rows include `decision` values of `pass`, `watch`, or `fail`.
- `turns scorecard-pack` writes JSON and Markdown scorecard artifacts using
  schema `tokensquash.turns.scorecard.pack.v1`.
- `turns compare-scorecards` reports corpus-size, savings, privacy, alias, and
  sidecar deltas using schema `tokensquash.turns.scorecard.compare.v1`.
- `turns scorecard-history` reports timeline and adjacent comparison evidence
  using schema `tokensquash.turns.scorecard.history.v1`.
- README and real-corpus docs show the scorecard command.
- Strict doctor and product manifest include the v0.2.0 plan, scorecard schema,
  scorecard pack schema, scorecard comparison schema, and scorecard history
  schema.

## Out Of Scope

- Replacing deterministic `ts1` or `tr1`.
- Claiming universal token savings.
- Treating local-AI sidecar output as canonical.
- Publishing private raw turns or unreviewed sidecar examples.
- PyPI publishing.
