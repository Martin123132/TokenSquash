# TokenSquash v0.1.1 Plan

`v0.1.1` should be a polish and evidence release. It should not change the core
idea or make broad token-saving claims. The goal is to make the existing
workflow easier to repeat, inspect, and explain.

## Release Theme

Make TokenSquash easier to evaluate from a fresh checkout:

- clearer roadmap and next-release scope
- less manual release asset documentation
- stronger first-real-corpus guidance
- better sidecar meaning-risk evidence
- small CLI/report polish where it removes friction

## Proposed Changes

### Release Evidence

- Use `release-assets --update-verification-doc` to generate the
  release-verification asset table from a staged `release-assets.json` report.
- Keep `release-assets --upload` explicit and review-first.
- Document the post-release update flow for changelog, release notes, asset
  hashes, and verification docs.

### Real Corpus Workflow

- Add a short first-run guide for collecting 10 real turns locally.
- Include the expected command sequence:
  `turns capture`, `turns report`, `turns suggestions`, `turns certify`, and
  optional `sidecar evaluate`.
- Make sure the guide says raw private turns stay ignored and local.

### Sidecar Evidence

- Improve sidecar review output so missing fields, generic decoded text,
  suspiciously short semantics, and warning counts are easier to scan.
- Add a small meaning-preservation rubric to the docs.
- Keep deterministic `ts1` and `tr1` unchanged.

### Documentation

- Keep README focused on first commands and link deeper details out to docs.
- Keep `ROADMAP.md`, `CHANGELOG.md`, and release docs in the strict doctor
  governance checks.
- Make commercial/non-commercial boundaries easy to find from the README.

## Acceptance Checklist

- `python -m unittest discover -s tests` passes.
- `python -m tokensquash doctor --strict` passes.
- `python -m tokensquash readiness --out-dir private-turns\readiness-v011 --json`
  passes.
- `python -m tokensquash verify-readiness private-turns\readiness-v011 --require-readiness-pass --json`
  passes.
- Any new release-facing command is listed by `python -m tokensquash about --json`.
- Public docs do not instruct users to commit raw private prompts, replies, or
  local model output.

## Out Of Scope

- changing `ts1` or `tr1` wire compatibility
- making sidecar semantic JSON canonical
- publishing to PyPI
- adding hosted services or cloud model dependencies
- claiming production-scale savings without real-corpus evidence

## Candidate Release Notes

`v0.1.1` improves the release and evaluation workflow around the existing
TokenSquash experiment. It adds clearer roadmap documentation, tightens
release-evidence guidance, and focuses the next measurement work on real local
turn corpora and sidecar meaning preservation. The deterministic codec remains
the source of truth, and the local-AI sidecar remains experimental.
